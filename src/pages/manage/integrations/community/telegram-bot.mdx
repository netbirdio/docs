# Telegram Bot for NetBird

A community-built Telegram bot that lets you manage your self-hosted NetBird network directly from Telegram — no dashboard needed.

**Repository:** [n0pashkov/netbird-tg-bot](https://github.com/n0pashkov/netbird-tg-bot)

## Features

- **Peers** — list, view details, approve pending peers, delete
- **Groups** — list, view, create, delete
- **Setup Keys** — list, view, create, revoke
- **Users** — list, view, invite, reinvite
- **Routes** — list, view
- **Policies** — list, view
- **Events** — recent activity log
- **⚙️ Settings** — configure Netbird connection, manage bot users, switch language — all from inside the bot

## Quick Start

### 1. Get a Telegram Bot Token

Open [@BotFather](https://t.me/BotFather) in Telegram, send `/newbot`, and copy the token.

### 2. Configure

```bash
cp .env.example .env
```

Edit `.env`:

```env
NB_TELEGRAM_TOKEN=your_telegram_bot_token
```

### 3. Run

```bash
docker compose up -d
```

### 4. First-run setup

Send `/start` to your bot. The first user becomes admin automatically and is guided through a setup wizard to configure the NetBird API connection:

1. Choose interface language (English or Russian)
2. Enter your NetBird API URL (e.g. `https://nb.example.com`)
3. Enter your NetBird API token (create one in **Dashboard → Settings → Access Tokens**)

## Access Roles

| Role | Permissions |
|------|-------------|
| `admin` | Full access — view, create, delete, manage bot settings |
| `viewer` | Read-only access |

Additional users are managed via **⚙️ Settings → Bot Users** inside the bot.

## Notes

- Written in Go, deployed as a single Docker container
- All settings are persisted in SQLite — no config files required
- Supports HTTP proxy via `HTTPS_PROXY` environment variable
